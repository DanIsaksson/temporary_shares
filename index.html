Core C# Concepts Explained

Understanding Classes

In C#, a class is a fundamental building block of object-oriented programming (OOP) that serves as a blueprint for creating objects. Objects are instances of a class, and they encapsulate data (attributes or properties) and behavior (methods or functions) that operate on that data. 

public class Player
{
    // Properties (data)
    public string Name { get; private set; }
    public int Health { get; private set; }

    // Constructor (initialization)
    public Player(string name)
    {
        Name = name;
        Health = 100;
    }
}

The Player class defines what a "Player" is in the context of a game project. It specifies that every Player object will have a Name (a string) and Health (an integer). The constructor demonstrates how classes can define the initial state of their objects.

Demystifying Properties: get and set

Properties in C# are members of a class that provide a flexible mechanism to read, write, or compute the value of a private field. They offer a level of abstraction over direct field access, allowing for validation, computation, or other logic to be executed when a value is retrieved or assigned.

get Accessor

The get accessor is used to return the property value. It allows external code to read the property's current value.

Console.WriteLine(player.Name);

// Reads the Name property

set Accessor

The set accessor is used to assign a new value. It allows external code to modify the property's value.

player.Name = "NewName";

// Sets the Name property

In the Player class example, public string Name { get; private set; } and public int Health { get; private set; } are auto-implemented properties. This means the compiler automatically creates a private, anonymous backing field that can only be accessed through the property's get and set accessors.

The get; private set; Pattern

The get; private set; pattern is a common and powerful C# idiom used to create read-only properties from outside the class, while still allowing the class itself to modify the property's value.

Key Benefits:

  • Encapsulation: Protects internal state from arbitrary external modification
  • Controlled Access: Ensures data integrity through class methods
  • Read-Only Exposure: Allows reading while restricting writing

When a property has a public get accessor, any other code in the application can read the current value of that property. The private set accessor means that only the code within the class can assign a new value to this property.

This is particularly useful for ensuring data integrity and enforcing business rules. For example, the Health property might have specific rules: it shouldn't be negative, or it shouldn't exceed a maximum value. By making the set accessor private, the Player class can implement methods like TakeDamage(int amount) that internally modify the Health property, applying any necessary logic before assigning it.

Methods: Adding Behavior to Classes

Methods are blocks of code within a class that define the actions or operations an object of that class can perform. They encapsulate behavior, allowing objects to do things, interact with their own data, or interact with other objects. 

public class Player
{
    // ... properties ...
    
    // Method to modify Health with validation
    public void TakeDamage(int amount)
    {
        if (amount > 0) // Ensure damage is positive
        {
            Health -= amount;
            if (Health < 0) Health = 0; // Health shouldn't be negative
        }
    }
}

In the Player class, the constructor public Player(string name) is a special type of method that initializes a new Player object. To change the Health of a Player (which has a private set), you would typically have a method within the Player class, such as TakeDamage.

This method would contain the logic for reducing the player's health, potentially checking if the health drops below zero, and triggering other game events. The Python example take_damage(self, amount) clearly demonstrates a method that modifies the internal state of an object, showcasing how methods provide controlled ways to change object data, especially when properties have restricted setters.

Interactive Playbook: The Player Class in Action

The Player Class: Code Walkthrough

The Player class serves as the central example for this interactive playbook. Let's break down its components to understand its structure and behavior: 

public class Player
{
    // Properties with get; private set;
    public string Name { get; private set; }  // Get makes it so any one code outside can read the player's name.
    public int Health { get; private set; }   // private set makes so Health can only be changed by code inside the Player class itself.

    // Constructor
    public Player(string name)
    {
        Name = name;
        Health = 100;
    }

    // Method to modify Health
    public void TakeDamage(int amount)
    {
        if (amount > 0) // Ensure damage is positive
        {
            Health -= amount;
            if (Health < 0) Health = 0; // Health shouldn't be negative
        }
    }

    // Method to display player info
    public void DisplayInfo()
    {
        Console.WriteLine($"Name: {Name}");
        Console.WriteLine($"Health: {Health}");
    }
}

Class Declaration

public class Player defines a new class named Player. The public access modifier means this class can be accessed from other parts of the program.

Properties

Properties define the data attributes of the class with controlled access:

  • public get: External code can read the value
  • private set: Only code within the class can modify the value

Constructor

public Player(string name) is called when a new Player object is created. It initializes the object's state with the provided name and default health of 100.

Methods

Methods define the behaviors and operations the class can perform:

  • TakeDamage: Modifies Health with validation logic
  • DisplayInfo: Shows player information
